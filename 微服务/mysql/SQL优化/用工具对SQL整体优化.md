做事要有方法论，要先整体后局部，解决问题要注重效率，先尽量考虑不改写的优化，再考虑改写的优化 而不改写的优化靠的是体系结构知识的沉淀，而改写则妥考虑、逻辑等价改写和业务改写两大思路，其中业务改写是 SQL 优化的最高境界。
